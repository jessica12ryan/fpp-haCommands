# Contributing

Pull requests and issues are welcome.

## Reporting Issues

Report bugs or feature requests at:

https://github.com/jessica12ryan/fpp-haCommands/issues

Please include:
- FPP version
- xLights version (if applicable)
- Home Assistant version
- Steps to reproduce
- Relevant logs from `/home/fpp/media/logs/fppd.log` or `config/debug.log`

If the issue is related to a major security concern that by posting on GitHub could open FPP instances to vulnerabilities, than we request that you email your concern to jessica12ryan@outlook.com so we can privately address it.

## Pull Requests

- Test changes on your FPP instance before submitting
- Match the existing code style (PHP 8, comments)
- Update documentation where necessary
- Keep scope focused — one change per PR

## Developer Installation

```bash
cd /home/fpp/media/plugins
git clone https://github.com/jessica12ryan/fpp-haCommands.git
bash fpp-haCommands/scripts/fpp_install.sh
```

Then enable the plugin in **FPP UI → Content Setup → Plugin Manager**.

## How It Works

### FPP Plugin Architecture

- `scripts/fpp_install.sh` — Plugin install hook; updates Apache CSP and sets executable permissions
- `api.php` — PHP backend for HA connection test, entity refresh, and command generation
- `commands/descriptions.json` — FPP command definitions generated by `api.php`
- `commands/call_action.php` — Script executed by FPP to call HA REST API services
- `commands/get_state.php` — Script executed by FPP to fetch entity state
- `config.php` — Plugin settings page (HA URL, token, test, refresh)
- `menu.inc` / `status.php` / `about.php` — FPP plugin UI pages

### xLights Sequence Integration

When FPP plays a sequence with embedded commands:
- At each frame that has an FPP command marker, FPP looks up the named **Command Preset**
- The preset fires the associated **HA command** (e.g., `HA - light.turn_on` → calls `call_action.php`)
- The HA REST API is called with the configured entity and parameters
- Commands fire at the exact frame — synced to your sequence timeline

You can verify execution in the FPP log:
```bash
grep "HA Commands" /home/fpp/media/logs/fppd.log
```
